## 1. Установка Ollama

**Linux/macOS:**
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

**Windows:**
Скачай установщик с [ollama.com](https://ollama.com/download)

**Проверка установки:**
```bash
ollama --version
```

---

## 2. Скачать модель Qwen2.5:0.5b

```bash
ollama pull qwen2.5:0.5b
```

**Проверить список скачанных моделей:**
```bash
ollama list
```

---

## 3. Поднять Ollama сервер

**Запуск сервера** (в отдельном терминале):
```bash
ollama serve
```

> По умолчанию сервер запускается на `http://localhost:11434`

---

## 4. Скрипт для отправки запросов

**Примеры использования**

```
python3 ollama_client.py -p "Что такое doker?"
python3 ollama_client.py -p "Объясни динамическое программирование" --stream
```

            <td>Вот пример кода на Python, который проверяет, является ли строка палиндромом:

```python
def is_palindrome(s):
    # Переводим каждую букву в нижнем регистре
    s = s.lower()
    
    # Инициализируем переменную для хранения итераций по строке
    count = 0
    
    # Возвращаем True, если строка палиндромом (содержит одинаковую часть)
    for i in range(1, len(s) + 1):
        # Удаляем пробелы и регистрируем символы в нижнем регистре
        substring = s[i - 1:i]
        if not substring or substring[0] == substring[-1]:
            count += 1
    
    return True if count == len(s) // 2 else False

# Пример использования функции:
input_str = "Madam, I'm Adam"
result = is_palindrome(input_str)
print(f"Строка '{input_str}' - палиндром: {result}")
```

Этот код делает следующее:

1. `is_palindrome` — это функция, которая принимает строку и проверяет, является ли она палиндромом.
2. Функция переводит все буквы в нижнем регистре и удалив пробелы.
3. В цикле проходит по каждой строке из исходной строки:
   - Удаляем пробелы.
   - Проверяем, что символы начинаются и заканчиваются на пустую строку (символы " ").
4. В случае совпадения размера срезов (сумма шагов) и уникальности символов в первом и втором срезах, функция вернет `True`.
5. Если это не палиндром, вернем `False`.

Этот код можно использовать для проверки различных строк и возвращать соответствующее сообщение об их палиндроме или нет.</td>

            <td>Проблема двух генералов в распределённой системе — это проблема при работе с несколькими операциями (диспансерными и другим типом операций), когда это вызывает проблемы с управлением нагрузкой и взаимодействием между этими операциями.

В примере можно предположить, что у нас есть два генерала: один генерал-диспансер и другой — другому типу операции. В данной ситуации, диспансер должен быть в состоянии выполнять определенные задачи (например, устранять проблемы на объекте), а другая группа — иметь право применить другие операции или давать рекомендации.

Указанный пример демонстрирует, что проблемы могут возникнуть из-за различных факторов, включая:

1. Несоответствие требований: Диспансер может не поддерживать необходимую нагрузочную структуру или может не иметь достаточно свободного времени для выполнения его задач.

2. Нарушение принципов работы: Диспансер мог бы упростить свою работу, но в данном случае это приведет к проблемам с управлением генералами-операторами и их взаимодействием.

3. Использование сложных механизмов: Для эффективного выполнения задач диспансера необходимо использовать специализированные механизмы, которые могут вызвать проблемы с управлением другими операциями.

4. Закрытие задач: Если одна операция может быть отключена или отсутствует в процессе решения задачи, это также может привести к проблемам.

5. Недостаточное управление: В некоторых системах часто возникают ситуации, когда несколько генералов-диспансера не имеют достаточного количества времени и устаревших механизмов для решения задачи.

Важно помнить, что в реальном мире распределённой системы все это могут быть более сложными ситуациями, так как многие системы требуют регулярных обновлений и пересмотра их взаимодействия.</td>

            <td>Почему в компьютерах используется двоичная система, а не десятичная?</td>
            <td>Вместе цифры и буквы являются одними из основных типов данных в компьютерной системе. В качестве системы счисления для хранения информации используются различные системы счисления, из-за которых могут быть различия в подходах к обработке данных.

Для начала стоит учесть, что десятичная система, известна как ASCII, используется для хранения и передачи текстовых сообщений. Она использует 7-bit (8-разрядные) символы: 0-127, представляющие собой 16 различных букв (от 'а' до 'z'), цифр (0-9), и пробел. Это очень удобно для обработки текстовой информации в различных языках.

В отличие от десятичной системы счисления, двоичная система занимает только 8-разрядные символы: 0-7, представляющие собой 16 различных букв (от 'а' до 'z'), цифр (0-9), и пробел. Это делает двоичную систему подходящей для обработки текстовых данных.

В качестве дополнительного фактора стоит отметить, что в ходе процесса извлечения числа из исходных данных можно использовать только 4 биты (8-разрядные), что упрощает работу с двоичными системами. Это делает их более эффективными и подходящими для обработки большого объема информации.

В конечном счете, выбор между двоичной и десятичной системами счисления зависит от конкретных требований к удобству хранения, передачи и обратной связи данных. ASCII-система является стандартом для общего использования в различных языках программирования, а ASCII-исчисление используется для обработки информации из форматов, таких как ZIP или ZIP7.</td>
